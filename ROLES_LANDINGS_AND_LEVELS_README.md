## Система уровней

```
🌱 Новичок   —   0 XP (старт)
🚀 Джуниор   —  50 XP (~5 откликов)
⚡ Миддл     — 200 XP (~20 откликов или 5 дней стрика с откликами)
💎 Сеньор    — 600 XP (~60 откликов или месяц стрика)
👑 Мастер    — 1500 XP
🔥 Легенда   — 4000 XP
```

XP начисляется автоматически:
- Отклик: +10 XP (+5 за «жирный» заказ с score ≥ 30)
- Посещение нового дня: +5 + бонус от стрика (1-10 XP)
- Первый визит: +10 XP

Константы в `lib/levels.js` — `LEVELS[].minXp`. Изменить пороги — одна правка.

## Автоприменение роли

Логика:

1. **SSR (в `app/page.js`):** если `profile.user_role` есть и в URL нет фильтров (`role`, `category`, `source`, `search`) — `initialProjects` фильтруется по роли. То есть роботы/боты Яндекса и Google видят полную ленту на `/`, но залогиненный юзер получает отфильтрованные данные в SSR.

2. **Клиент (в `ProjectsFeed`):** при монтировании, если у юзера есть `user_role`, URL чистый и в `sessionStorage` нет флага `role_dismissed` — делается `router.replace('/?role=X')`. URL становится `/?role=X`, чтобы клиентский запрос данных совпал со SSR.

3. **Откат:** при клике «Сбросить фильтры» в Sidebar ставится `sessionStorage.role_dismissed = '1'` — в этой сессии роль больше автоматически не применяется. После следующего захода (новая сессия) — снова применяется.

4. **Смена роли в настройках:** при сохранении сбрасывается `sessionStorage.role_dismissed` → следующий заход применит новую роль.

Это решает UX-дилемму: по умолчанию пользователь видит «своё» (релевантнее), но может вручную расширить до всех проектов и сайт это запомнит до конца сессии.

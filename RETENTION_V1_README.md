# Retention v1: промежуточный экран + exit intent + избранное

Реализует первый уровень удержания из нашего обсуждения:

1. **Промежуточный экран** `/projects/[id]/responded` — при клике «Перейти» биржа открывается в новой вкладке, а **текущая** вкладка переходит на follow-up экран с таймером, похожими заказами и кнопками отметить результат. Когда юзер возвращается — он видит не пустую ленту, а persist-экран со следующими шагами.

2. **Exit Intent модалка** — для анонимов, когда мышь уходит к верхней границе браузера. Предлагает оставить email для еженедельного дайджеста. Накопление лидов вместо потери трафика.

3. **Избранное + отметка результата** — залогиненный пользователь может сохранить проект в избранное, отметить «Получил ответ» / «Взяли в работу». За это начисляется XP. Это превращает случайный клик в полноценное действие и копит данные о юзере.

## Новые файлы

- `supabase/schema-v9-retention.sql` — расширяет `applications` (status, status_updated_at, deal_amount, notes), добавляет `saved_projects`, `email_captures`, `viewed_projects`

## Как это выглядит в UX

**Сценарий 1: залогиненный юзер, клик «Перейти»**

1. Биржа открывается в новой вкладке
2. Текущая вкладка мгновенно переходит на `/projects/[id]/responded`
3. Юзер видит экран с зелёной галочкой «Отправили тебя на биржу»
4. Таймер 20 мин («столько пишется качественный отклик»)
5. Идёт писать отклик в соседней вкладке
6. Через 10 минут возвращается в нашу вкладку — там уже:
   - Заголовок «С возвращением!»
   - 4 кнопки «Отправил / Ответил / Взяли в работу / Отказ»
   - Кнопка «Сохранить в избранное»
   - 6 похожих заказов ниже — можно сразу открыть следующий
7. Клик на похожий → открывается новая вкладка с биржей → наша вкладка снова на responded-экране
8. **Петля удержания**: каждый отклик → follow-up → похожие → отклик

**Сценарий 2: аноним, клик «Перейти»**

То же самое, но вместо статусов показывается CTA «Зарегистрируйся и считай свою конверсию». Перехватываем момент когда юзер уже совершил действие (отклик) и имеет мотивацию создать аккаунт.

**Сценарий 3: аноним, уходит не откликнувшись**

Через 20 сек на сайте, при движении мыши к верхней границе → модалка «Оставь email для еженедельного дайджеста топ-10 заказов». Показывается **один раз за сессию**.

## Метрики для отслеживания

После деплоя мониторь в Яндекс Метрике/Google Analytics:

- **Просмотры `/projects/[id]/responded`** — сколько юзеров доходит до follow-up экрана
- **Bounce rate на responded**: сколько сразу закрывает. Если > 70% — значит не возвращаются с биржи, нужно думать дальше
- **Клики «Сохранить в избранное» и «Отметить результат»** — engagement метрики
- **Клики на похожие проекты** — основной KPI этого экрана
- **Отправки email-capture** (в БД `email_captures`) — конверсия exit intent

Хороший ориентир: конверсия клика на похожий проект → 10-15%. Регистрация из exit intent → 3-5%. Отметка статуса → 20-30% от залогиненных.

## Настройки под капотом

**Таймер на follow-up экране** — 20 минут. Менять в `components/RespondedClient.js`:
```js
const [remaining, setRemaining] = useState(20 * 60);
```

**Exit intent триггеры** — 20 секунд на сайте + движение мыши в верхние 10px. Менять в `components/ExitIntent.js`:
```js
const MIN_TIME_MS = 20 * 1000;
```

**XP за статусы** — в `app/api/applications/update-status/route.js`:
- +20 XP за «responded»
- +50 XP за «accepted»

**Rate-limit email-capture** — 5 попыток с одного IP за час.
В `app/api/email-capture/route.js`.

## Важно

- Промежуточный экран помечен `robots: { index: false }` — не попадёт в поиск. Хорошо.
- Exit intent модалка помнит своё состояние в `sessionStorage` → показывается один раз за вкладку, при закрытии и открытии снова — может появиться опять (это норм).
- `email_captures.ip_hash` — это SHA256 от IP + CRON_SECRET, сам IP не хранится. GDPR/ФЗ-152 дружелюбно.

## Что дальше (следующие шаги retention-плана)

Сейчас это **первый уровень** — remediate момент ухода. Дальше по плану:

1. **Дневной персональный дайджест в Telegram-боте** — самый жирный win. Раз в день бот шлёт лично «5 заказов с 80%+ совпадения за ночь»
2. **Страница `/me/applications`** — личная история откликов с конверсией, возможность редактировать статусы задним числом
3. **Страница `/me/saved`** — избранные проекты
4. **Email-реанимация через 3 дня неактивности** — «пока тебя не было, появилось X жирных заказов»
5. **Скрытие просмотренных** — «Скрыть уже виденные» чекбокс в ленте, использует таблицу `viewed_projects`
6. **AI-анализ «почему не отвечают»** — Claude разбирает последние отклики юзера и даёт персональную рекомендацию

Первый (Telegram-дайджест) — самый высокий ROI за усилия. Скажи когда готов — сделаю.
